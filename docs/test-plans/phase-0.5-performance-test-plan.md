# Phase 0.5 性能优化测试计划

> **测试目标**: 验证移动端性能优化功能在各种设备上的效果
> **测试周期**: 2026-03-10 ~ 2026-03-12（3 天）
> **负责人**: QA Team + Frontend Developer

## 测试场景

### 场景 A：简单场景

**主题**: "牛顿第一定律"

**预期内容**:
- 1-2 个几何体
- 无粒子系统或少量粒子（< 100）
- 简单动画

**预期性能**:
- 所有设备 FPS >= 30

### 场景 B：中等场景

**主题**: "太阳系行星运动"

**预期内容**:
- 5-8 个几何体（太阳 + 行星）
- 中等粒子系统（1000-5000）
- 轨道动画

**预期性能**:
- 高性能设备: FPS >= 60
- 中等性能设备: FPS >= 45
- 低性能设备: FPS >= 30

### 场景 C：复杂场景

**主题**: "量子力学波函数"

**预期内容**:
- 10+ 个几何体
- 大量粒子系统（5000-10000）
- 复杂动画和后处理

**预期性能**:
- 高性能设备: FPS >= 60
- 中等性能设备: FPS >= 45
- 低性能设备: FPS >= 30（自动降级后）

## 测试用例

### TC-001: 设备性能检测

**目的**: 验证设备性能检测功能是否正确

**步骤**:
1. 在测试设备上打开任意 UPG 生成的可视化
2. 打开浏览器控制台
3. 查看 "Device Performance" 日志

**预期结果**:
- 日志显示正确的设备信息（tier, deviceType, gpuScore, memory, cores）
- 性能等级符合预期（见测试设备清单）

**验收标准**:
- ✅ 所有设备的性能等级判断准确率 >= 80%

### TC-002: 自适应渲染配置

**目的**: 验证不同性能等级使用不同的渲染配置

**步骤**:
1. 在高性能设备上打开场景 C
2. 查看控制台 "Render Config" 日志
3. 验证配置参数（antialias, pixelRatio, maxParticles 等）
4. 在低性能设备上重复步骤 1-3

**预期结果**:
- 高性能设备: antialias=true, maxParticles=10000
- 低性能设备: antialias=false, maxParticles=1000

**验收标准**:
- ✅ 渲染配置根据性能等级正确调整

### TC-003: FPS 监控

**目的**: 验证 FPS 监控功能是否正常工作

**步骤**:
1. 在测试设备上打开场景 B
2. 观察右上角的 FPS 显示
3. 记录 FPS 值和颜色
4. 快速拖拽、缩放场景，观察 FPS 变化

**预期结果**:
- FPS 显示实时更新（每秒更新一次）
- FPS 颜色根据帧率变化：
  - 绿色: >= 目标 FPS 的 90%
  - 黄色: >= 目标 FPS 的 70%
  - 红色: < 目标 FPS 的 70%

**验收标准**:
- ✅ FPS 显示准确
- ✅ 颜色变化正确

### TC-004: 性能模式切换

**目的**: 验证用户可以手动切换性能模式

**步骤**:
1. 在测试设备上打开场景 B
2. 点击右上角的性能模式下拉菜单
3. 选择"高性能"
4. 确认刷新页面
5. 观察 FPS 和画质变化
6. 重复步骤 2-5，选择"省电模式"

**预期结果**:
- 切换到高性能: FPS 提升，画质更好（有抗锯齿）
- 切换到省电模式: FPS 稳定，画质降低（无抗锯齿）
- 设置保存到 localStorage

**验收标准**:
- ✅ 性能模式切换功能正常
- ✅ 设置持久化保存

### TC-005: 自适应降级

**目的**: 验证 FPS 过低时自动降级功能

**步骤**:
1. 在低性能设备上打开场景 C
2. 观察初始 FPS
3. 等待 3-5 秒，观察 FPS 是否提升
4. 查看控制台是否有 "Low FPS detected, reducing quality..." 日志

**预期结果**:
- 如果初始 FPS < 目标 FPS 的 70%，系统自动降级
- 降级后 FPS 提升
- 控制台显示降级日志

**验收标准**:
- ✅ 自适应降级功能正常工作
- ✅ 降级后 FPS 提升至少 20%

### TC-006: 移动端触摸优化

**目的**: 验证移动端触摸手势和防滚动功能

**步骤**:
1. 在移动设备上打开场景 B
2. 单指拖动场景，验证旋转功能
3. 双指缩放场景，验证缩放功能
4. 双指拖动场景，验证平移功能
5. 尝试在场景上滚动页面，验证防滚动功能
6. 观察是否显示触摸提示（3秒后淡出）

**预期结果**:
- 单指拖动: 场景旋转，页面不滚动
- 双指缩放: 场景缩放，页面不缩放
- 双指拖动: 场景平移
- 触摸提示显示并在 3 秒后淡出

**验收标准**:
- ✅ 所有触摸手势正常工作
- ✅ 页面滚动冲突已解决
- ✅ 触摸提示正常显示

### TC-007: 长时间运行稳定性

**目的**: 验证长时间运行不会崩溃或内存泄漏

**步骤**:
1. 在测试设备上打开场景 B
2. 让场景运行 5 分钟
3. 每分钟记录一次 FPS 和内存占用
4. 观察是否有崩溃或卡顿

**预期结果**:
- 5 分钟内 FPS 保持稳定（波动 < 10%）
- 内存占用稳定（增长 < 20%）
- 无崩溃或卡顿

**验收标准**:
- ✅ 长时间运行稳定
- ✅ 无内存泄漏

### TC-008: 跨浏览器兼容性

**目的**: 验证性能优化在不同浏览器上都能正常工作

**步骤**:
1. 在 Safari 上打开场景 B，记录 FPS
2. 在 Chrome 上打开场景 B，记录 FPS
3. 对比两个浏览器的性能差异

**预期结果**:
- 两个浏览器的 FPS 差异 < 20%
- 性能优化功能在两个浏览器上都正常工作

**验收标准**:
- ✅ 跨浏览器兼容性良好

### TC-009: 网络状态检测

**目的**: 验证网络状态检测功能

**步骤**:
1. 在测试设备上打开场景 B（在线状态）
2. 查看控制台 "Device Performance" 日志中的网络状态
3. 切换到飞行模式
4. 刷新页面，查看网络状态

**预期结果**:
- 在线状态: isOnline=true, connectionType='fast' 或 'slow'
- 离线状态: isOnline=false, connectionType='offline'

**验收标准**:
- ✅ 网络状态检测准确

### TC-010: 电池优化模式

**目的**: 验证省电模式下的电池优化效果

**步骤**:
1. 在移动设备上打开场景 B
2. 切换到省电模式
3. 运行 10 分钟，记录电池消耗
4. 切换到高性能模式
5. 运行 10 分钟，记录电池消耗
6. 对比两种模式的电池消耗

**预期结果**:
- 省电模式的电池消耗比高性能模式低至少 30%

**验收标准**:
- ✅ 省电模式有效降低电池消耗
