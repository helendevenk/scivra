# 北美 K-12 科学实验需求总规划文档

**项目**: NeonPhysics v2 - UPG 万物原理生成器
**目标市场**: 北美学生（K-12，5-18岁）
**文档版本**: v1.0
**创建日期**: 2026-03-09
**调研范围**: NGSS 标准 + AP 课程 + PhET 等主流平台分析

## 执行摘要

本文档基于 NGSS（Next Generation Science Standards）和 AP 课程标准，系统梳理了北美小学（K-5）、初中（6-8）、高中（9-12）三个学段的科学实验需求。共识别出 **120+ 个适合可视化/交互式模拟的实验主题**，按优先级分为三级：

- **P0（必须有）**: 28 个核心实验，覆盖 NGSS 核心标准和 AP 考试重点
- **P1（强烈建议）**: 42 个重要实验，增强课程完整性
- **P2（锦上添花）**: 50+ 个扩展实验，提供差异化学习体验

**关键洞察**：
1. **物理学科最强需求**：占比 54.5%，是可视化模拟的最佳领域
2. **初高中是主战场**：6-12 年级学生对交互式模拟的需求最强烈
3. **市场空白**：现有平台（PhET、Gizmos）缺乏零依赖、AI 生成、移动优先的解决方案
4. **商业潜力**：北美 K-12 STEM 教育市场规模 $8B+，在线模拟工具渗透率 < 30%

### 1.3 小学实验设计原则

1. **简化物理模型**：避免复杂公式，用动画和比喻解释
2. **游戏化设计**：挑战关卡、即时反馈、成就徽章
3. **安全第一**：虚拟实验替代危险操作（如火焰、化学品）
4. **跨学科整合**：结合数学（测量、图表）和语言艺术（观察记录）
5. **移动优先**：支持平板和 Chromebook（小学主流设备）

### 8.2 NeonPhysics v2 的差异化优势

#### 核心差异化（3个独特卖点）

1. **零依赖 HTML + AI 生成**
   - **问题**：现有平台需要网络、登录、插件，学校网络环境受限
   - **解决方案**：生成完全自包含的 HTML 文件，可离线使用、嵌入任何网站
   - **价值**：最快的加载速度（< 1秒）、最广的兼容性（任何浏览器）、最低的技术门槛

2. **AI 驱动的个性化实验**
   - **问题**：现有平台实验固定，无法满足教师的个性化需求
   - **解决方案**：教师可用自然语言描述需求，AI 生成定制实验
   - **价值**：无限的实验变体、快速迭代、适应不同教学风格

3. **移动优先 + 中英双语**
   - **问题**：现有平台多为桌面优先，移动体验差；缺乏中文支持
   - **解决方案**：响应式设计、触摸优化、中英文无缝切换
   - **价值**：服务北美华裔学生（200万+）、国际学校、BYOD 课堂

#### 次要差异化（5个竞争优势）

4. **开源友好**：生成的 HTML 可下载、修改、分享（类似 CodePen）
5. **社区驱动**：教师可分享自定义实验到画廊，形成内容生态
6. **嵌入式设计**：一键生成嵌入代码，集成到教师网站/LMS
7. **Freemium 模式**：核心功能免费，降低采用门槛
8. **快速迭代**：AI 生成 → 每周新增实验，而非传统的年度更新

### 10.5 关键决策点

**决策 1：MVP 范围确认**
- **问题**：10 个实验是否足够验证市场需求？
- **选项 A**：10 个实验（快速验证，3个月）
- **选项 B**：20 个实验（更完整，6个月）
- **推荐**：选项 A（快速验证，降低风险）

**决策 2：技术栈选择**
- **问题**：是否需要引入物理引擎库（如 Matter.js）？
- **选项 A**：自研轻量级物理引擎（零依赖，但开发成本高）
- **选项 B**：使用 Matter.js 简化版（依赖外部库，但开发快）
- **推荐**：选项 A（符合零依赖原则，长期价值高）

**决策 3：定价策略**
- **问题**：是否应该完全免费（通过广告/赞助盈利）？
- **选项 A**：Freemium（免费 + 付费订阅）
- **选项 B**：完全免费（广告/赞助）
- **推荐**：选项 A（可持续商业模式，避免广告干扰学习体验）

**决策 4：市场优先级**
- **问题**：是否应该同时进入小学、初中、高中市场？
- **选项 A**：聚焦高中（AP 考试需求强，付费意愿高）
- **选项 B**：全学段覆盖（K-12）
- **推荐**：选项 B（全学段覆盖，最大化市场规模）

### 10.6 风险与应对

**风险 1：AI 生成质量不稳定**
- **影响**：生成的实验不符合教学标准，教师不信任
- **应对**：建立实验模板库 + 质量检查机制 + 人工审核

**风险 2：竞品快速跟进**
- **影响**：PhET、Gizmos 推出类似功能，失去差异化优势
- **应对**：专注零依赖 + AI 生成 + 社区驱动，建立护城河

**风险 3：教师采用门槛高**
- **影响**：教师不愿意尝试新工具，增长缓慢
- **应对**：提供详细教程 + 免费培训工作坊 + 教师社区支持

**风险 4：技术债务积累**
- **影响**：快速开发导致代码质量下降，维护成本高
- **应对**：建立代码审查机制 + 单元测试 + 定期重构

**风险 5：资金不足**
- **影响**：无法支撑长期开发，项目中断
- **应对**：申请教育科技基金 + 天使投资 + 众筹
