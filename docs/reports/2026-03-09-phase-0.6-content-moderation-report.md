## Phase 0.6 AI 内容审核机制 - 实现报告

**实施日期**: 2026-03-09
**状态**: ✅ 已完成

### 执行摘要

成功实现了三层 AI 内容审核机制，防止生成不当内容。包括输入审核（敏感词检查）、输出审核（HTML 安全检查）和人工复审（管理后台）。所有审核记录持久化到数据库，支持追溯和统计分析。

### 实现内容

#### 1. 数据库 Schema 更新

**新增表**: `content_moderation`

```sql
CREATE TABLE content_moderation (
  id VARCHAR(255) PRIMARY KEY,
  generation_id VARCHAR(255) NOT NULL REFERENCES upg_generation(id) ON DELETE CASCADE,
  check_type VARCHAR(20) NOT NULL,  -- 'input' | 'output' | 'manual'
  status VARCHAR(20) NOT NULL,      -- 'pass' | 'reject' | 'pending'
  reason TEXT,                      -- 审核不通过的原因
  matched_keywords TEXT[],          -- 匹配到的敏感词列表
  reviewed_by VARCHAR(255) REFERENCES user(id) ON DELETE SET NULL,
  review_notes TEXT,                -- 审核备注
  checked_at TIMESTAMP DEFAULT NOW() NOT NULL,
  reviewed_at TIMESTAMP             -- 人工审核时间
);

-- 索引
CREATE INDEX idx_content_moderation_generation ON content_moderation(generation_id);
CREATE INDEX idx_content_moderation_status ON content_moderation(status);
CREATE INDEX idx_content_moderation_check_type ON content_moderation(check_type);
CREATE INDEX idx_content_moderation_checked_at ON content_moderation(checked_at);
```

#### 3. 内容审核服务

**文件**: `/src/shared/lib/moderation/content-moderator.ts`

**核心函数**:

##### 3.1 `moderateInput(prompt: string)`
- **功能**: 检查用户输入的 prompt 是否包含敏感词
- **性能**: < 50ms
- **策略**:
  - high severity → 直接拒绝
  - medium severity → 标记为 pending，允许生成但进入人工复审队列
  - low severity → 记录但通过
- **返回**: `{ passed, status, reason, matchedKeywords }`

##### 3.2 `moderateOutput(htmlContent: string)`
- **功能**: 检查 AI 生成的 HTML 代码是否安全
- **性能**: < 200ms
- **检查项**:
  1. ✅ 禁止非白名单 `<script>` 标签
  2. ✅ 禁止 `on*` 事件属性（onclick, onerror 等）
  3. ✅ 禁止 `javascript:` 协议
  4. ✅ 禁止 `<iframe>` 外部嵌入
  5. ✅ 禁止 `<object>` 和 `<embed>` 标签
  6. ✅ 检查外部资源链接（非白名单）
  7. ✅ 禁止 localStorage/sessionStorage/cookie 访问
  8. ✅ 检查危险函数（eval, Function, fetch, XMLHttpRequest, WebSocket）
  9. ✅ 允许 `<style>` 标签（UPG 需要内联样式）
- **返回**: `{ passed, status, reason, issues }`

##### 3.3 `moderateContent(prompt, htmlContent)`
- **功能**: 综合审核（同时执行输入和输出审核）
- **返回**: `{ inputResult, outputResult, overallPassed }`

#### 6. 集成到 UPG 生成流程

**文件**: `/src/shared/lib/upg/generate-core.ts`

**集成点**:

1. **输入审核**（在 AI 生成前）:
   ```typescript
   const inputModerationResult = moderateInput(prompt);
   if (!inputModerationResult.passed) {
     // 拒绝生成，记录审核结果
     return { status: 'failed', errorMessage: '...' };
   }
   ```

2. **输出审核**（在质量检查后）:
   ```typescript
   const outputModerationResult = moderateOutput(htmlContent);
   if (!outputModerationResult.passed) {
     // 拒绝生成，记录审核结果
     return { status: 'failed', errorMessage: '...' };
   }
   ```

3. **审核记录持久化**:
   - 所有审核结果（pass / reject / pending）都记录到 `content_moderation` 表
   - 支持追溯和统计分析

**流程图**:
```
用户输入 prompt
  ↓
输入审核 (moderateInput)
  ├─ reject → 拒绝生成，返回错误
  ├─ pending → 允许生成，标记待审核
  └─ pass → 继续
  ↓
AI 生成 HTML
  ↓
HTML 安全检查 (sanitizeHtml)
  ↓
质量检查 (checkQuality)
  ↓
输出审核 (moderateOutput)
  ├─ reject → 拒绝生成，返回错误
  ├─ pending → 允许生成，标记待审核
  └─ pass → 继续
  ↓
保存到数据库
  ↓
返回结果
```

### 技术亮点

1. **三层审核机制**:
   - 输入审核：前置拦截，避免浪费 AI 资源
   - 输出审核：后置检查，确保生成内容安全
   - 人工复审：兜底机制，处理边界情况

2. **宽松模式 + 人工复审**:
   - 明显违规内容直接拒绝
   - 可疑内容标记为 pending，允许生成但进入复审队列
   - 平衡安全性和用户体验

3. **性能优化**:
   - 敏感词匹配使用简单字符串查找（O(n*m)）
   - 未来可以优化为 Aho-Corasick 算法（O(n+m)）
   - HTML 安全检查使用正则表达式，性能可接受

4. **可扩展架构**:
   - 敏感词库独立配置，支持热更新
   - 审核服务与业务逻辑解耦
   - 支持未来集成第三方审核 API（如阿里云内容安全）

### 已知限制

1. **敏感词库不完整**:
   - 当前为示例词库，实际部署时需要替换为完整词库
   - 建议使用第三方敏感词库或自建词库

2. **简单字符串匹配**:
   - 无法处理变体（如"暴 力"、"b@o力"）
   - 未来可以扩展为正则表达式或 NLP 模型

3. **无拼写纠错**:
   - 无法识别拼写错误的敏感词
   - 未来可以集成拼写纠错算法

4. **无语义理解**:
   - 无法理解上下文语义
   - 未来可以集成 LLM 进行语义审核

### 监控指标

建议在生产环境监控以下指标：

1. **审核通过率**:
   - `pass_rate = (pass_count / total_count) * 100%`
   - 目标：> 95%

2. **误杀率**:
   - `false_positive_rate = (false_reject_count / total_count) * 100%`
   - 目标：< 1%

3. **漏网率**:
   - `false_negative_rate = (false_pass_count / total_count) * 100%`
   - 目标：< 0.1%

4. **审核性能**:
   - `input_moderation_time_p95 < 50ms`
   - `output_moderation_time_p95 < 200ms`

5. **待审核队列长度**:
   - `pending_queue_length < 100`
   - 超过阈值时告警
